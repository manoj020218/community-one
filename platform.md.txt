Jenix Community One 

Codex Task: Build Jenix Society One — Core Platform First

You are a senior full-stack product architect and software engineer.

Build a professional, scalable, 10-year-life SaaS product foundation called:

Jenix Society One

This is a common modular society-management platform for residential societies, apartments, gated communities and townships.

The platform must be designed so that future modules can be added phase by phase as plugin-type modules without rewriting the core.

Future modules will include:

1. Parking + Boom Barrier + UHF Reader
2. Visitor Management
3. Maintenance Collection + Receipt
4. Fund Utilization + Transparent Audit
5. Facility Booking for Common Hall, Party Hall, Swimming Pool, Club, Movie Hall etc.
6. Member Access Control for Gym, Spa, Pool
7. Announcement + Events + Fundraising
8. Complaint Management
9. Staff / Maid / Vendor Management
10. Delivery Management
11. Emergency / SOS
12. Polling / Voting
13. Document Management
14. Asset / AMC Management
15. IoT Device Health Monitoring

But in this first development cycle, build only the Core Platform.

The core platform must be strong enough that all future modules can be plugged in easily.

Very Important Engineering Rules

Follow these strictly.

1. No file should be more than 200 lines.
2. Split large components, services, routes, controllers and schemas into smaller files.
3. Use modular folder structure.
4. Every phase must be tested against written unit tests.
5. After completing one phase, run tests.
6. If tests fail, fix them before moving to next phase.
7. Work in a loop:
   - Plan the phase
   - Create files
   - Write tests
   - Run tests
   - Fix errors
   - Run tests again
   - Mark phase complete only after tests pass
8. Do not write dummy untested code.
9. Do not create fake “complete” status unless tests pass.
10. Keep code production-quality and easy to debug.
11. Every important backend action must be audit-log ready.
12. Every future module must be easy to register in the module registry.
13. Every API must respect "societyId" isolation.
14. Design as multi-tenant SaaS from day one.
15. Add watchdog-friendly architecture for long-term product health.

Jenix Society One is not only a society app. It is a modular operating system for societies.

Each society will get its own login and isolated data.

Example:

The platform should work like a SaaS product where Jenix Super Admin creates a society, enables modules, creates society admin, and society admin completes onboarding.
